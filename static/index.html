Chrome PDF bug reproduction

This page will have set two (session) cookies:

  • Auth1=auth1; SameSite=Lax
  • Auth2=auth2

There are three (large) files here, which will only be served if the cookies are present. The bug means that Chrome doesn't send the SameSite=Lax cookie on a Range request in the Chrome PDF viewer, so the PDF only partially renders. The image and video are unaffected.
